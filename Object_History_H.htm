Object History V

Use this page to Track Object History.

To access this page:

  1. Select dspMigrate in the Navigation pane, or select Console in the Context bar.
  2. Select Elements > Object in the Navigation pane.
  3. Click the History icon for an Object.

Field

Description

ID

Displays the Object identity. Changes are tracked in ascending order, so the higher the number the more recent the change.

OBJECT NAME

Displays the name of the Object.

Wave

Displays the name of the Wave that the Object is assigned to.

Process Area

Displays the name of the Process Area that the Object is assigned to.

PRIORITY

Displays the Object‚Äôs priority, which is the order the Object displays on the Objects page‚Äôs Horizontal View.

OBJECT OWNER

Displays the name of the user registered in the platform who is responsible for the Object.

OBJECT STATUS

Displays the overall status of the migration Object, such as In Development, Deferred, or Completed.

TARGET SYSTEM

Displays the name of the Target system.

TARGET TABLES

Displays the Target tables used in the Object.

T CODE

Displays the SAP Transaction Code or the Target System Type transaction code.

VERSION

Displays the version of the Object. ERP systems allows different versions for recordings.

Object History H

Field

Description

ID

Displays the Object identity. Changes are tracked in ascending order, so the higher the number the more recent the change.

Object Name

Displays the name of the Object.

Description

Displays a brief description of the Object.

Wave

Displays the name of the Wave that the Object is assigned to.

Process Area

Displays the name of the Process Area that the Object is assigned to.

Priority

Displays the Object‚Äôs priority, which is the order the Object displays on the Objects page‚Äôs Horizontal View.

Data Type

Displays the type of data represented by the Object. Options include Master Data, Ref/Conditional, and Transactional.

Content Driver

Displays the name of the group that is driving the contents. Options include Company, Enterprise, and Legal.

Spec

Click Upload to upload the specifications document to the web server. If a specifications document has been uploaded, the download icon is available. Click Download to download the document from the web server.

Comment

Displays a user-entered comment about the Object.

Load Type

Displays the method used to load the Object during a Mock Wave. Options include Automatic, Manual or ALE.

RICEFW

Displays a RICEFW number. RICEFW is a generic industry term for "Reports, Interfaces, Conversions, Extensions, Forms and Workflow.‚Äù In the overall Project Plan, Deliverables are often assigned a RICEFW number to identify and track the development progress of that item.  

Conversion Objects are a deliverable, so therefore they may be assigned a RICEFW number by the PMO.

Complexity

Displays the complexity for building the Object. This value can be used to evaluate work assignments. Options include Average, Simple or Complex. These options are set on the List Value Configuration page and can be edited. If a Target is created in Map that does not have an assigned Complexity, this value will be passed to Transform to complete the required Complexity field on Transform's Targets page‚Äôs Vertical View on the Documentation tab. A user can then update the complexity in Transform.

Approx Volume

Displays an estimate of the number of records to load for the Object.

Legacy Systems

Displays the user-entered legacy systems.

History Requirements

Displays the user-entered history requirements.

Notes

Displays user-entered notes about the Object.
