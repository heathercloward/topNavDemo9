Configure Mapping Status and Rule Status

Map tracks the Mapping Status and Rule Status of each mapping.

There are two types of Mapping Status and Rule Status.

Installed Mapping Statuses and Rule Statuses cannot be deleted. All attributes can be configured except for the name. Installed statuses are applied during the Mapping Approval process and indicate where a mapping or rule is in the process.

The system changes these statuses when the mapping is submitted, when it has been reviewed and approved, or when it has been reviewed and rejected.

A user can change the Mapping Status on the Vertical View of the Field Mappings page to an installed status or a custom status.

A user cannot update the Rule Status in Map. It is updated by the system.

The user can update the Rule Status in SQL AutoGen. Refer to Update a Rule Status in SQL AutoGen for more information.

Refer to Mapping Status and Rule Status for detailed information about installed statuses.

Custom statuses can be added to track work on mapping and rule creation. These statuses are applied manually and are not set by the system. Custom statuses, such as "Impeded," are used for documentation purposes as an additional status beyond installed statuses for tracking mapping progress.

A custom status is assigned to mappings between the status of Pending Review and Complete. It cannot be used to define a mapping status beyond Complete.

NOTE: Custom statuses display on the Target Source Rule Book, but do not display on dashboards.

To add a custom Mapping Status in Map:

  1. Select Configuration > Mapping Status in the Navigation pane.

  2. Click Add.

    View the field descriptions for the Mapping Status page

  3. Enter the name of the status in the MAPPING STATUS field.
  4. Enter a brief description of the status in the DESCRIPTION field.

  5. Select an image from the IMAGE NAME list box.

    NOTE: Images that were installed with the system display. These images can be viewed on the Images page (Admin > Configuration > Style > Images). A user can also add custom images on this page.

  6. Click the DEFAULT check box to enable it, if applicable.

    NOTE: The default status is set for a mapping when the mapping is saved. The system uses the In Progress status unless another status has this setting enabled.

  7. Click the TRACK HISTORY check box to enable it, if applicable.

    NOTE: If the check box is enabled, when a mapping is set to this status, it is tracked on the Field Mapping History page.

  8. Click Save.

This custom Mapping Status displays as an option in the Mapping Status list box on the Field Mappings pageâ€™s Vertical View.

To configure a custom Rule Status:

  1. Select Configuration > Rule Status in the Navigation pane.

  2. Click Add.

    View the field descriptions for the Rule Status page.

  3. Enter the name of the status in the RULE STATUS field.
  4. Enter a brief description of the status in the DESCRIPTION field.

  5. Select an image from the IMAGE NAME list box.

    NOTE: Images that were installed with the system display. These images can be viewed on the Images page (Admin > Configuration > Style > Images). A user can also add custom images on this page.

  6. Click Save.

This custom Rule Status displays as an option in the RULE STATUS list box on the Automation SQL Field Mappings page.
