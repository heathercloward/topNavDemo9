Import Field Mappings

Use this page to Import Mappings.

To access this page in dspMigrate:

  1. Select Map in the Context bar.
  2. Click the Targets icon on the Process Area Launch page.
  3. Click the Sources icon for a Target.
  4. Click the Vertical View icon for the Source.
  5. Click the Import Field Mappings icon in the Page toolbar.

Field

Description

Import Mappings

Click to import mappings from the System Type.

ERP TABLE NAME

Displays ERP Target table name. For example, if the Target table is ttMARA, the ERP Target table name is MARA.

SOURCE DATABASE OBJECT

Displays name of Source database object used in the mapping. For example, for an SAP to SAP mapping, the Source database object could be MARA to MARA. For a JDE to SAP mapping, the Source database object could be F0411 to MARA.

TARGET FIELD

Displays field within Target table being mapped.

FIELD GROUP

Displays field group to which the selected field belongs. Field groups are assigned in Target Design, and allow users to apply filters for mapping based on certain values in a field.

MAPPING STATUS

Displays the Mapping Status of the field in the mapping process.

Users may not want to import Completed mappings.

RULE STATUS

Displays the Rule Status of the rule to be built from the mapping once it is complete.

Users may not want to import Completed mappings.

ACTION

Displays current action of field mapping.

ACTION NEW

Displays new action being imported from System Type Mappings.

DEFAULT VALUE

Displays current default value of field mapping.

DEFAULT VALUE NEW

Displays new default value being imported from System Type Mappings.

SOURCE TABLE

Displays current Source table of field mapping.

SOURCE TABLE NEW

Displays new Source table being imported from System Type Mappings.

SOURCE FIELD

Displays current Source field of field mapping.

SOURCE FIELD NEW

Displays new Source table being imported from System Type Mappings.

RULE SQL

Displays current rule SQL of field mapping.

RULE SQL NEW

Displays new rule SQL being imported from System Type Mappings.

RULES COMMENT

Displays current rule comment on field mapping.

RULES COMMENT NEW

Displays new rule comment being imported from System Type Mappings.

MAPPED BY

Displays user ID of individual who set the current mapping status.

MAPPED ON

Displays date and time when the MAPPING STATUS was set on the Field Mappings page.

SUBMITTED BY

Displays the user ID of the user who clicked Submit on the Field Mappings page.

SUBMITTED ON

Displays date and time when the Submit button was clicked on the Field Mappings page.

APPROVED BY

Displays user ID of Developer who approved the mapping on the Mapping Approval page.

APPROVED ON

Displays date and time when a Developer approved the mapping on the Mapping Approval page.

TARGET DESCRIPTION

Displays a brief description of the Target table.

MAPPING SET

Displays the name of the mapping set, which is a categorization of mappings that are imported and exported.

TARGET RELATIONSHIP

Displays the name of the relationship, for example: Join MARA to MARC (Material to Plants).

TARGET RELATIONSHIP NEW

Displays the name of the relationship being imported from System Type Mappings.
